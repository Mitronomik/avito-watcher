# Google Sheets alerts with LLM summary runbook

This runbook describes the production alert contour for Google Sheets alerts with optional LLM scoring data. It is an operator runbook only: it does not change runtime defaults, enable email, or make the LLM part of the core monitoring loop.

Production contour:

1. Worker monitors Avito search/feed URLs.
2. JSONL stays enabled as the local audit trail.
3. Google Sheets receives an external alert row through a webhook.
4. LLM scoring may run after rule filters. When scoring succeeds and `LLM_SHADOW_MODE=false`, the Google Sheets row includes `summary`, `score`, and `tags`.
5. Analytics are out of scope for this runbook.

## Non-goals and safety boundaries

- Do not enable email for this contour. Keep `EMAIL_ENABLED=false` and keep `email` out of `ALERT_CHANNELS`.
- Do not change production defaults just to run smoke checks. Use one-off environment overrides on `docker compose run`.
- Do not disable JSONL. Keep `jsonl` in `ALERT_CHANNELS` and keep `JSONL_OUTBOX_ENABLED=true` so every attempted production alert leaves a local audit record.
- Do not expect alerts during first baseline initialization. Smoke checks must use a search whose baseline is already initialized and that can produce a new listing alert.
- Keep LLM fail-soft. If scoring fails, alert delivery must continue without LLM fields.

## Required Google Apps Script behavior

The Google Apps Script Web App must return JSON exactly shaped as an object with `ok: true` on success:

```json
{"ok": true}
```

If the script returns non-JSON, `ok: false`, or JSON without `ok: true`, the Google Sheets notifier treats the delivery as failed. Error responses should also be JSON and must not echo secrets:

```json
{"ok": false, "error": "Unauthorized"}
```

## Required secret behavior

- Set a shared secret in Google Apps Script Script Properties as `WEBHOOK_SECRET`.
- Pass the same value to the app as `GOOGLE_SHEETS_WEBHOOK_SECRET` through deployment secret storage or an interactive shell variable for a one-off smoke command.
- Never hardcode real secrets in repository files, examples, screenshots, shell history, or spreadsheet formulas.
- The webhook payload includes `secret` only so Apps Script can authenticate the request. Do not append this field to the sheet row.
- Rotate the secret before go-live, after accidental disclosure, and when operator access changes.

## Expected Google Sheets columns

Create or validate the header row in this exact order:

```text
sent_at, search_name, external_id, title, price, area_m2, rooms, address,
published_label, published_at, url, summary, score, tags, message
```

LLM fields behave as follows:

- `summary`, `score`, and `tags` are blank when scoring is disabled, the provider is `off`, scoring fails, or `LLM_SHADOW_MODE=true`.
- `summary`, `score`, and `tags` are populated only when scoring succeeds and `LLM_SHADOW_MODE=false`.
- `tags` may arrive as an array; Apps Script should serialize it to a comma-separated string for the sheet cell.

For the current commercial rent production contour, use the commercial-rent LLM prompt profile:

```text
LLM_PROMPT_PROFILE=commercial_rent
```

Supported prompt profiles are `commercial_rent`, `flat_sale`, `flat_rent`, and `generic_real_estate`. Keep `commercial_rent` for commercial lease monitoring; use a different profile only when the same scoring path is intentionally reused for another real estate category.

## Recommended freshness filters for commercial rent

When enabling a new alert channel such as `google_sheets`, existing listings may be pending for that channel even though they are not fresh publications. For commercial real estate rent monitoring, prefer an explicit freshness policy in `SearchJob.filters_json`:

```json
{
  "min_area_m2": 40,
  "max_area_m2": 150,
  "max_price": 200000,
  "require_published_at": true,
  "max_age_hours": 72,
  "missing_published_at_policy": "reject",
  "source_sort": "date",
  "exclude_keywords": ["кладовка", "паркинг", "машиноместо", "гараж"]
}
```

Operator notes:

- First baseline initialization is intentionally silent and does not send alerts.
- `max_age_hours` limits alerts to listings with parsed publication time inside the freshness window.
- `missing_published_at_policy=reject` is safest when freshness matters; use `allow` or `allow_when_date_sorted` only after accepting the stale-publication risk.
- If `missing_published_at_policy=allow_when_date_sorted` is used, the source Avito URL should be sorted by publication date and `source_sort` should be set to `date`.

## Payload contract

The Google Sheets webhook receives a JSON object with these fields:

| Field | Required | Notes |
| --- | --- | --- |
| `secret` | yes | Shared secret for Apps Script validation; do not write to the sheet. |
| `sent_at` | yes | UTC timestamp generated by the notifier. |
| `search_name` | yes | Search job name. |
| `external_id` | yes | Avito listing external ID. |
| `title` | yes | Listing title from the search/feed card. |
| `price` | no | Numeric price when parsed. |
| `area_m2` | no | Numeric area when parsed. |
| `rooms` | no | Room count/label when parsed. |
| `address` | no | Address text from the search/feed card. |
| `published_label` | no | Human publication label from the card. |
| `published_at` | no | Parsed publication timestamp when available. |
| `url` | yes | Listing URL. |
| `summary` | no | LLM summary only when included in the alert path. |
| `score` | no | LLM score only when included in the alert path. |
| `tags` | no | LLM tags only when included in the alert path. |
| `message` | yes | Human-readable alert message. |

## Safe Apps Script reference implementation

Use this as a minimal Web App receiver or compare it with the maintained script in `docs/integrations/google_sheets_apps_script.gs`. Store `WEBHOOK_SECRET` in Script Properties before production use. This implementation assumes a spreadsheet-bound Apps Script because it calls `SpreadsheetApp.getActiveSpreadsheet()`.

If you deploy a standalone Apps Script project, adapt the script to read a `SPREADSHEET_ID` Script Property and open the target spreadsheet with `SpreadsheetApp.openById(...)` before selecting or creating the alert sheet.

```javascript
const DEFAULT_SHEET_NAME = 'Alerts';

const COLUMNS = [
  'sent_at',
  'search_name',
  'external_id',
  'title',
  'price',
  'area_m2',
  'rooms',
  'address',
  'published_label',
  'published_at',
  'url',
  'summary',
  'score',
  'tags',
  'message',
];

function doPost(e) {
  try {
    const payload = parsePayload_(e);
    validateSecret_(payload.secret);

    const spreadsheet = SpreadsheetApp.getActiveSpreadsheet();
    const sheetName = getScriptProperty_('SHEET_NAME') || DEFAULT_SHEET_NAME;
    const sheet = spreadsheet.getSheetByName(sheetName) || spreadsheet.insertSheet(sheetName);

    ensureHeader_(sheet);
    sheet.appendRow(mapRow_(payload));

    return jsonResponse_({ ok: true });
  } catch (err) {
    return jsonResponse_({ ok: false, error: safeError_(err) });
  }
}

function parsePayload_(e) {
  if (!e || !e.postData || !e.postData.contents) {
    throw new Error('Missing request body');
  }
  return JSON.parse(e.postData.contents);
}

function validateSecret_(providedSecret) {
  const expectedSecret = getScriptProperty_('WEBHOOK_SECRET');
  if (!expectedSecret) {
    throw new Error('WEBHOOK_SECRET is not configured');
  }
  if (!providedSecret || providedSecret !== expectedSecret) {
    throw new Error('Unauthorized');
  }
}

function getScriptProperty_(name) {
  return PropertiesService.getScriptProperties().getProperty(name);
}

function ensureHeader_(sheet) {
  if (sheet.getLastRow() === 0) {
    sheet.appendRow(COLUMNS);
    return;
  }

  const firstRow = sheet.getRange(1, 1, 1, COLUMNS.length).getValues()[0];
  const missingHeaders = COLUMNS.some((name, index) => firstRow[index] !== name);
  if (missingHeaders) {
    sheet.insertRowBefore(1);
    sheet.getRange(1, 1, 1, COLUMNS.length).setValues([COLUMNS]);
  }
}

function mapRow_(payload) {
  return [
    valueOrEmpty_(payload.sent_at),
    valueOrEmpty_(payload.search_name),
    valueOrEmpty_(payload.external_id),
    valueOrEmpty_(payload.title),
    valueOrEmpty_(payload.price),
    valueOrEmpty_(payload.area_m2),
    valueOrEmpty_(payload.rooms),
    valueOrEmpty_(payload.address),
    valueOrEmpty_(payload.published_label),
    valueOrEmpty_(payload.published_at),
    valueOrEmpty_(payload.url),
    valueOrEmpty_(payload.summary),
    valueOrEmpty_(payload.score),
    normalizeTags_(payload.tags),
    valueOrEmpty_(payload.message),
  ];
}

function normalizeTags_(tags) {
  if (Array.isArray(tags)) {
    return tags.join(', ');
  }
  return valueOrEmpty_(tags);
}

function valueOrEmpty_(value) {
  if (value === null || value === undefined) {
    return '';
  }
  return String(value);
}

function jsonResponse_(obj) {
  return ContentService
    .createTextOutput(JSON.stringify(obj))
    .setMimeType(ContentService.MimeType.JSON);
}

function safeError_(err) {
  if (!err || !err.message) {
    return 'Unknown error';
  }
  return err.message;
}
```

## Smoke data and dedupe strategy

Each smoke check that expects a new Google Sheets row needs a listing whose `external_id` is pending for the `google_sheets` channel. Alert delivery is deduplicated with per-channel alert records, so after Google Sheets delivery succeeds for an `external_id`, rerunning `run-once` for that same already-delivered listing will not append another Google Sheets row.

Read the `run-once` result before treating a smoke as successful:

- If `created=0`, no new listing snapshot was created and the smoke may not have produced any pending alert.
- If `delivery_attempted_by_channel.google_sheets=0`, Google Sheets delivery was not actually exercised.
- The "Google Sheets channel without LLM" smoke and the "LLM included in Google Sheets alert" smoke should not rely on the same already-delivered `external_id` if the operator expects two separate Google Sheets rows.

Recommended safe options:

- Use a temporary smoke `SearchJob` and wait for a genuinely new listing.
- Re-run smoke only when another new listing appears for the selected search.
- Otherwise ensure there is a new or pending alert for the `google_sheets` channel before expecting a row.

Do not manually delete production `alert_sent` rows to force re-delivery unless there is a verified backup and an explicit operator decision to mutate production alert history.

## One-off smoke: Google Sheets channel without LLM

This checks JSONL plus Google Sheets delivery without editing production `.env` and without enabling scoring.

```bash
export GOOGLE_SHEETS_WEBHOOK_URL='https://script.google.com/macros/s/<deployment-id>/exec'
read -r -s GOOGLE_SHEETS_WEBHOOK_SECRET
export GOOGLE_SHEETS_WEBHOOK_SECRET

docker compose --env-file .env -f deploy/docker-compose.prod.yml run --rm \
  -e ALERT_CHANNELS=jsonl,google_sheets \
  -e JSONL_OUTBOX_ENABLED=true \
  -e GOOGLE_SHEETS_WEBHOOK_ENABLED=true \
  -e GOOGLE_SHEETS_WEBHOOK_URL \
  -e GOOGLE_SHEETS_WEBHOOK_SECRET \
  -e SCORING_ENABLED=false \
  -e LLM_PROVIDER=off \
  -e LLM_SHADOW_MODE=true \
  app python3 -m app.cli run-once --search-id <ID>
```

Expected result:

- `run-once` exits successfully.
- A new JSONL alert line appears at `JSONL_OUTBOX_PATH`.
- A Google Sheets row is appended.
- LLM columns are blank.
- The Apps Script response is `{"ok": true}`.

## One-off smoke: LLM shadow mode

This checks that scoring can run and persist its result for history while keeping LLM fields out of alert rows.

```bash
read -r -s LLM_API_KEY
export LLM_API_KEY

docker compose --env-file .env -f deploy/docker-compose.prod.yml run --rm \
  -e ALERT_CHANNELS=jsonl,google_sheets \
  -e JSONL_OUTBOX_ENABLED=true \
  -e GOOGLE_SHEETS_WEBHOOK_ENABLED=true \
  -e GOOGLE_SHEETS_WEBHOOK_URL \
  -e GOOGLE_SHEETS_WEBHOOK_SECRET \
  -e SCORING_ENABLED=true \
  -e LLM_SHADOW_MODE=true \
  -e LLM_PROVIDER=openai_compatible \
  -e LLM_BASE_URL=https://api.deepseek.com \
  -e LLM_MODEL=<model> \
  -e LLM_PROMPT_PROFILE=commercial_rent \
  -e LLM_API_KEY \
  app python3 -m app.cli run-once --search-id <ID>
```

Expected result:

- JSONL and Google Sheets delivery still work.
- The selected listing snapshot stores LLM scoring metadata in history when scoring succeeds.
- Google Sheets `summary`, `score`, and `tags` remain blank because shadow mode is enabled.

## Rollback to non-LLM JSONL-only alerts

Use one-off rollback verification first:

```bash
docker compose --env-file .env -f deploy/docker-compose.prod.yml run --rm \
  -e ALERT_CHANNELS=jsonl \
  -e JSONL_OUTBOX_ENABLED=true \
  -e GOOGLE_SHEETS_WEBHOOK_ENABLED=false \
  -e SCORING_ENABLED=false \
  -e LLM_PROVIDER=off \
  -e LLM_SHADOW_MODE=true \
  app python3 -m app.cli run-once --search-id <ID>
```

Then return production configuration to the safe contour:

```env
ALERT_CHANNELS=jsonl
JSONL_OUTBOX_ENABLED=true
GOOGLE_SHEETS_WEBHOOK_ENABLED=false
SCORING_ENABLED=false
LLM_PROVIDER=off
LLM_SHADOW_MODE=true
EMAIL_ENABLED=false
```

If the worker is running, restart it after changing deployed secrets/configuration so it picks up the rollback values:

```bash
docker compose --env-file .env -f deploy/docker-compose.prod.yml --profile worker restart worker
```
